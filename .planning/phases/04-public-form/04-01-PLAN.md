  <action>Create Zod schema file for issue reporting validation:

Schema fields:
- reporterName: z.string().min(2).max(100)
- reporterEmail: z.string().email()
- reporterPhone: z.string().regex(/^[+]?[\d\s-]{9,15}$/)
- title: z.string().min(5).max(200)
- description: z.string().min(20).max(2000)
- type: z.enum(['annotation', 'comment', 'accident'])
- priority: z.enum(['low', 'medium', 'high', 'critical'])
- photos: z.array(z.object({ id: z.string(), url: z.string(), caption: z.string().optional() })).optional()
- location: z.object({
    building: z.string().optional(),
    floor: z.string().optional(),
    area: z.string().optional()
  }).optional()
- verificationCode: z.string().length(6)
- qrSlug: z.string().min(3)
- accessToken: z.string().uuid()

Follow pattern from app/schemas/qr.ts (named exports, default barrel export).

<task type="auto">
  <name>Task 5: Create public issue submission API endpoint</name>
  <files>server/api/public/issue-report/submit.post.ts</files>
  <action>Create POST endpoint at /api/public/issue-report/submit:

Endpoint logic:
1. Validate request body with IssueReportSchema (Zod)
2. Verify QR access by calling validateIssueQRAccess from services/issueQRService.ts (reuses existing validation)
3. Verify verification code exists and is not expired (query VerificationCode model)
4. Upload photos to S3 using existing S3 upload pattern (or collect URLs if pre-uploaded)
5. Generate reference number using generateReferenceNumber()
6. Create Issue document with:
   - obraId from QR validation
   - Title, description, type, priority from form
   - Photos array with uploaded URLs
   - Status: "open", Priority: "medium" (or from form)
   - createdBy: "public-reporter" (special marker)
   - Additional metadata: reporter info (name, email, phone - stored in comments or separate field)
7. Mark verification code as used (verified: true)
8. Return IssueReportResponse with reference number and created issue

Error handling:
- 400: Validation errors with details
- 403: Invalid QR access or verification code
- 410: Expired QR or verification code
- 500: Server errors with logging

Follow server/api/public/issue-report/[planId]/[slug].get.ts error pattern (createError with statusCode, statusMessage, message).
Console log for debugging (following existing pattern).
