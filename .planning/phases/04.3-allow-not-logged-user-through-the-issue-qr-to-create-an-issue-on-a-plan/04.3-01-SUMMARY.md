## Decisions Made

### Key Decision: Public QR Access Without Token Validation

**Rationale:** QR codes are placed at construction sites for public access. Anyone (workers, visitors, inspectors) should be able to scan and report issues. The QR slug only identifies which obra/work the issue belongs to. No access control is needed for issue reporting.

**Impact:**
- `accessToken` in URL is now ignored - only slug matters
- Public endpoints validate: slug exists, QR enabled, not expired
- Anyone can access `/public/issues/{qrSlug}/{any-token}`

### Schema Decision: Passthrough vs Strict

**Rationale:** Zod's `.strict()` mode was causing h3's `readValidatedBody` to fail with "fn is not a function" error. Changed to `.passthrough()` to allow extra fields while validating required ones.

**Impact:** Schema is more permissive, handles extra fields gracefully, prevents validation errors.

### Color Compatibility: Nuxt UI v4

**Rationale:** Old color names (`gray`, `blue`) are not valid in Nuxt UI v4. Updated to new color names (`neutral`, `info`).

**Impact:** Components render correctly without TypeScript errors.
