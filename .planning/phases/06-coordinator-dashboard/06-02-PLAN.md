  <name>Task 1: Create public issues list endpoint</name>
  <files>server/api/public/issues/index.get.ts</files>
  <action>
    Create GET /api/public/issues endpoint that returns issues for an obra with QR token validation:

    1. Extract query params: obraId (required), qrToken (required)

    2. Validate QR token using issueQRService.validateIssueQRAccess(obraId, qrToken)
       - Return 403 if validation fails

    3. Query Issue model: Issue.find({ obraId }).lean()
       - Sort by createdAt: -1 (newest first)

    4. Return safe data only (map to MiniIssue format):
       - id, title, type, status, priority, createdAt, photoCount, commentCount
       - Exclude sensitive fields like createdBy, internal notes

    Follow the pattern from server/api/issues/[id].get.ts for error handling.
  </action>
  <verify>
    - Valid QR token returns list of issues for the obra
    - Invalid QR token returns 403 Forbidden
    - Missing obraId returns 400 Bad Request
    - Response contains only public-safe fields
    - Empty obra returns empty array (not 404)
  </verify>
  <done>Public issues list endpoint with QR token validation</done>
</task>

<task type="auto">
  <name>Task 2: Create public issue detail endpoint</name>
  <files>server/api/public/issues/[id].get.ts</files>
  <action>
    Create GET /api/public/issues/[id] endpoint that returns a single issue with QR token validation:

    1. Extract params: id (route param), qrToken (query param)

    2. Find issue by id: Issue.findById(id).lean()

    3. Return 404 if issue not found

    4. Validate QR token using issueQRService.validateIssueQRAccess(issue.obraId, qrToken)
       - Return 403 if validation fails

    5. Return full issue data with safe filtering:
       - Include: title, description, type, status, priority, photos, comments, createdAt, updatedAt
       - Transform photos and comments to include display names instead of internal IDs
       - Exclude: internal fields, sensitive user data

    Use Zod validation for query params if desired (optional but recommended).
  </action>
  <verify>
    - Valid QR token returns full issue details
    - Invalid QR token returns 403 Forbidden
    - Non-existent issue returns 404 Not Found
    - Comments include userName for display (not userId)
    - Photos include all metadata needed for display
  </verify>
  <done>Public issue detail endpoint with QR token validation</done>
</task>
