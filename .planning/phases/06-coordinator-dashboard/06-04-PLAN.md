       - schema: { text: z.string().min(1).max(2000) }

    5. Push to issue.comments array

    7. Return the new comment object

    Follow the existing Issue model comment structure.
  </action>

  </verify>

<task type="auto">
  <name>Task 2: Create comment deletion endpoint</name>
  <files>server/api/issues/[id]/comments/[commentId].delete.ts</files>
  <action>
    Create DELETE /api/issues/[id]/comments/[commentId] endpoint for removing comments:

    1. Check authentication

    2. Find issue by id, return 404 if not found

    3. Find comment by commentId in issue.comments array

    4. Return 404 if comment not found

    5. Check ownership: comment.userId === event.context.user._id
       - Also allow admins (event.context.user.role === 'admin')
       - Return 403 if not owner and not admin

    6. Remove comment from array using filter or pull

    7. Update issue.updatedAt = now()

    8. Return success response

    Use MongoDB's $pull operator for efficient array removal.
  </action>
  <verify>
    - Comment author can delete their own comment
    - Admin can delete any comment
    - Non-author non-admin returns 403
    - Deleted comment removed from array
    - Non-existent comment returns 404
  </verify>
  <done>Comment deletion endpoint with ownership check</done>
