<task type="auto">
  <name>Task 2: Create POST /api/subscriptions endpoint</name>
  <files>server/api/subscriptions/post.ts</files>
  <action>
Create POST endpoint at server/api/subscriptions/post.ts to create subscription:

1. Import Zod, createQrSubscription from utils/stripe, Subscription, Planes, User models
2. Verify authentication via event.context.user (401 if missing)
3. Extract userId from authenticated user
4. Parse and validate body with Zod schema:
   - planId: string required
   - billingInterval: enum('month', 'year') required
   - paymentMethodId: string required (Stripe payment method ID)
5. Verify plan exists and belongs to user (404 if not)
6. Check for existing active subscription on this plan (400 if exists)
   - Query: { planId, userId, status: { $in: ['active', 'past_due'] } }
7. Get user record, retrieve or create Stripe customer
8. Call createQrSubscription helper function from utils/stripe:
   - Pass: { customerId, planId, billingInterval, paymentMethodId, metadata: { userId, planId, billingInterval } }
   - Helper handles payment method attachment, default setting, and annual discount coupon
9. Save Subscription to database with:
    - All Stripe-mapped fields from response
    - annualPrepaymentDiscount: 15 if yearly, undefined otherwise
10. Return { success: true, data: { subscriptionId, stripeSubscriptionId, status, clientSecret } }

Follow server/api/payments/create-intent.post.ts pattern.
Note: This endpoint uses the createQrSubscription helper from Plan 03 (07-03) which encapsulates all Stripe subscription creation logic including payment method attachment and annual discount handling.
