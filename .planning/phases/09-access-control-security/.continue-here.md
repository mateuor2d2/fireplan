<context>

**What We're Building:**

Phase 9 implements the **pay-to-print** access control layer:
- Users can create and fill plans without payment
- PDF generation endpoint checks Payment.status === 'succeeded'
- Print button triggers payment flow if no payment exists
- QR issue reporting requires active subscription

**Key Technical Changes:**

1. **PDF Endpoint Modification** (Wave 1):
   - server/api/planes/[id]/generate-pdf.get.ts
   - Add Payment.findOne({ planId: id, status: 'succeeded' })
   - Return 403 if no payment, allow if payment exists

2. **Print Button Composable** (Wave 2):
   - app/composables/usePaymentGuard.ts
   - checkPayment(), requirePaymentOrTrigger()
   - Triggers payment flow with return URL

3. **Subscription Enforcement** (Wave 2):
   - server/api/planes/[id]/issues.post.ts
   - Check Subscription.status in ['active', 'past_due']
   - Return 403 if no active subscription

4. **CSP Headers** (Wave 3):
   - nuxt.config.ts updates for Stripe domains
   - js.stripe.com, checkout.stripe.com whitelisted

**User Flow After Implementation:**
```
1. Click "New Plan" → Go to form (FREE)
2. Fill out plan → Save (FREE)
3. Click "Print" → Check payment
   ├─ No payment → Stripe Checkout → Return → Print
   └─ Has payment → Print directly
```

- New approach is endpoint-based with print trigger
- All planning docs updated to reflect this
- Ready to execute with fresh context
