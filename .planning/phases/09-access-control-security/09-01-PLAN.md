2. Import Payment model if not already imported
3. After authentication check and plan retrieval, add payment enforcement:
   - Query Payment collection for planId with status === 'succeeded'
   - If no succeeded payment found: throw createError with:
     * statusCode: 403
     * statusMessage: "Payment required for PDF generation"
     * data: { paymentUrl: "/payment" } (or Stripe checkout URL when available)
   - If succeeded payment exists: continue with PDF generation

4. Implementation (add after plan retrieval):
   ```typescript
   import { Payment } from '~/models/Payment'

   // Check payment for PDF generation
   const payment = await Payment.findOne({
     planId: id,
     status: 'succeeded'
   })

   if (!payment) {
     throw createError({
       statusCode: 403,
       statusMessage: 'Payment required for PDF generation',
       data: {
         paymentUrl: `/payment?planId=${id}`,
         reason: 'no_payment_for_pdf',
         planId: id
       }
     })
   }
   ```

5. Handle cases where:
   - Payment exists but status is 'pending' or 'failed' → 403 with message
   - No payment found for this plan → 403 with payment URL
   - Payment is 'succeeded' → allow PDF generation

6. Preserve existing PDF generation logic after payment check
7. Add proper TypeScript types
