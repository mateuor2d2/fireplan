# Plan 10-01: Payment Status Endpoint

**Phase:** 10 Frontend Integration
**Wave:** 1 - API Layer
**Created:** 2026-01-27
**Status:** Pending

## Overview

Create a server API endpoint that returns payment status for a specific plan. This endpoint will be called by the `usePaymentGuard` composable to check if a user has paid for a plan before allowing PDF generation.

## Requirements

Covered: STAT-03 (partial - provides data for visual badges)

## Objective

Create `GET /api/payments/status/[planId]` endpoint that:
1. Returns payment status for the authenticated user's plan
2. Validates ownership (userId must match)
3. Returns structured data for UI consumption
4. Handles edge cases (no payment, multiple payments, pending status)

## Success Criteria

1. Endpoint returns 200 OK with payment status data
2. Returns 404 if plan exists but no payment found
3. Returns 403 if user doesn't own the plan
4. Returns structured JSON: `{ hasPayment, status, paymentId, amount, createdAt }`
5. Spanish error messages for validation failures

## Implementation

### File: `server/api/payments/status/[planId].get.ts`

```typescript
import { Payment } from '~/server/models/Payment'
import { getUserById } from '~/server/utils/userUtils'

/**
 * GET /api/payments/status/[planId]
 *
 * Returns payment status for a plan belonging to the authenticated user.
 *
 * Query Params:
 * - planId: string - The plan ID to check payment for
 *
 * Response:
 * {
 *   hasPayment: boolean
 *   status: 'none' | 'pending' | 'succeeded' | 'failed'
 *   paymentId?: string
 *   amount?: number
 *   createdAt?: string
 * }
 */

// Zod schema for planId validation
const PlanStatusQuerySchema = z.object({
  planId: z.string().min(1, 'El ID del plan es requerido')
})

export default defineEventHandler(async (event) => {
  try {
    // Get authenticated user from session
    const session = await getUserSession(event)
    if (!session?.userId) {
      throw createError({
        statusCode: 401,
        statusMessage: 'No autorizado: Inicia sesión para continuar'
      })
    }

    // Validate planId from route params
    const planId = getRouterParam(event, 'planId')
    if (!planId) {
      throw createError({
        statusCode: 400,
        statusMessage: 'El ID del plan es requerido'
      })
    }

    // Verify plan exists and user has access
    const plan = await Plan.findOne({ _id: planId, userId: session.userId })
    if (!plan) {
      throw createError({
        statusCode: 403,
        statusMessage: 'No tienes acceso a este plan'
      })
    }

    // Find payment for this plan (prefer succeeded, fallback to latest)
    const payment = await Payment.findOne({
      userId: session.userId,
      planId: planId
    }).sort({ createdAt: -1 })

    if (!payment) {
      // No payment found
      return {
        hasPayment: false,
        status: 'none'
      }
    }

    // Return payment status
    return {
      hasPayment: payment.status === 'succeeded',
      status: payment.status,
      paymentId: payment._id.toString(),
      amount: payment.amount,
      createdAt: payment.createdAt.toISOString()
    }
  } catch (error) {
    console.error('Error fetching payment status:', error)
    throw createError({
      statusCode: 500,
      statusMessage: 'Error al obtener el estado del pago'
    })
  }
})
```

### Key Implementation Notes

1. **Route Pattern**: Uses Nuxt dynamic route `[planId]` for RESTful design
2. **Ownership Validation**: Checks both userId in Payment and Plan
3. **Status Priority**: Returns latest payment if multiple exist
4. **Response Structure**: Designed for easy frontend consumption
5. **Error Messages**: Spanish for consistency with existing codebase
