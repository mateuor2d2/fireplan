# Plan 10-02: Stripe Checkout Session Creation

**Phase:** 10 Frontend Integration
**Wave:** 1 - API Layer
**Created:** 2026-01-27
**Status:** Pending

## Overview

Create a server API endpoint that creates Stripe Checkout sessions for one-time plan payments. This endpoint will be called by the payment page to initiate the hosted checkout flow.

## Requirements

Covered:
- PAY-03: Embedded Stripe Checkout session for pay-per-plan transaction
- PAY-04: Checkout session includes plan metadata (user, plan name)
- PAY-05: Success URL redirects back to plan page

## Objective

Create `POST /api/payments/create-checkout` endpoint that:
1. Creates or retrieves Stripe customer for the user
2. Creates a Stripe Checkout Session with line items
3. Creates a pending Payment record in database
4. Returns checkout URL for client-side redirection
5. Includes plan metadata in checkout session

## Success Criteria

1. Endpoint creates Stripe Checkout Session successfully
2. Pending Payment record created with stripePaymentIntentId
3. Checkout session includes plan name and metadata
4. Success URL points to print page with ?payment=success
5. Cancel URL points back to plan or custom return URL
6. Returns `{ checkoutUrl, sessionId }` for client redirection

## Implementation

### File: `server/api/payments/create-checkout.post.ts`

```typescript
import { Payment } from '~/server/models/Payment'
import { Plan } from '~/server/models/Plan'
import Stripe from 'stripe'
import { getUserById } from '~/server/utils/userUtils'

/**
 * POST /api/payments/create-checkout
 *
 * Creates a Stripe Checkout session for one-time plan payment.
 *
 * Body:
 * {
 *   planId: string
 *   planName?: string
 *   returnUrl?: string
 * }
 *
 * Response:
 * {
 *   checkoutUrl: string
 *   sessionId: string
 * }
 */

// Zod schema for request body
const CreateCheckoutSchema = z.object({
  planId: z.string().min(1, 'El ID del plan es requerido'),
  planName: z.string().optional(),
  returnUrl: z.string().optional()
})

// Fixed price per plan (€29.00 in cents)
// TODO: Make this configurable via admin settings in future
const PLAN_PRICE_EUR = 2900

const stripe = new Stripe(process.env.STRIPE_SECRET_KEY as string, {
  apiVersion: '2025-08-27.basil',
  typescript: true
})

export default defineEventHandler(async (event) => {
  try {
    // Get authenticated user
    const session = await getUserSession(event)
    if (!session?.userId) {
      throw createError({
        statusCode: 401,
        statusMessage: 'No autorizado: Inicia sesión para continuar'
      })
    }

    // Parse and validate request body
    const body = await readBody(event)
    const { planId, planName, returnUrl } = CreateCheckoutSchema.parse(body)

    // Verify plan exists and user has access
    const plan = await Plan.findOne({ _id: planId, userId: session.userId })
    if (!plan) {
      throw createError({
        statusCode: 403,
        statusMessage: 'No tienes acceso a este plan'
      })
    }

    // Check for existing succeeded payment (prevent double payment)
    const existingPayment = await Payment.findOne({
      userId: session.userId,
      planId: planId,
      status: 'succeeded'
    })

    if (existingPayment) {
      throw createError({
        statusCode: 400,
        statusMessage: 'Este plan ya ha sido pagado'
      })
    }

    // Get or create Stripe customer
    const user = await getUserById(session.userId)
    let stripeCustomerId = user?.stripeCustomerId

    if (!stripeCustomerId) {
      // Create new Stripe customer
      const customer = await stripe.customers.create({
        email: user.email,
        name: user.name,
        metadata: {
          userId: session.userId
        }
      })
      stripeCustomerId = customer.id

      // Save stripeCustomerId to user (TODO: update user model)
    }

    // Get plan name for checkout display
    const displayName = planName || plan.nom_obra || 'Plan de Seguridad'

    // Determine success and cancel URLs
    const siteUrl = process.env.NUXT_PUBLIC_SITE_URL || 'http://localhost:3000'
    const successUrl = `${siteUrl}/protected/planes/${planId}/print?payment=success`
    const cancelUrl = returnUrl || `${siteUrl}/protected/planes/${planId}`

    // Create Stripe Checkout Session
    const checkoutSession = await stripe.checkout.sessions.create({
      mode: 'payment',
      customer: stripeCustomerId,
      line_items: [{
        price_data: {
          currency: 'eur',
          product_data: {
            name: `Plan de Seguridad: ${displayName}`,
            description: 'Generación de PDF de Plan de Seguridad',
            metadata: {
              planId,
              planName: displayName
            }
          },
          unit_amount: PLAN_PRICE_EUR
        },
        quantity: 1
      }],
      success_url: successUrl,
      cancel_url: cancelUrl,
      metadata: {
        userId: session.userId,
        planId: planId,
        planName: displayName
      }
    })

    // Create pending Payment record
    // Note: PaymentIntent ID will be updated by webhook handler
    const payment = await Payment.create({
      userId: session.userId,
      planId: planId,
      stripePaymentIntentId: checkoutSession.payment_intent as string,
      amount: PLAN_PRICE_EUR,
      currency: 'eur',
      status: 'pending',
      description: `Pago del plan: ${displayName}`,
      metadata: {
        checkoutSessionId: checkoutSession.id,
        planName: displayName
      }
    })

    return {
      checkoutUrl: checkoutSession.url,
      sessionId: checkoutSession.id,
      paymentId: payment._id.toString()
    }
  } catch (error) {
    console.error('Error creating checkout session:', error)

    // Handle Zod validation errors
    if (error instanceof z.ZodError) {
      throw createError({
        statusCode: 400,
        statusMessage: 'Datos inválidos',
        data: error.errors
      })
    }

    throw createError({
      statusCode: 500,
      statusMessage: 'Error al crear la sesión de pago'
    })
  }
})
```

### Key Implementation Notes

1. **Double Payment Prevention**: Checks for existing 'succeeded' payment
2. **Customer Management**: Creates or reuses Stripe customer
3. **Metadata Tracking**: Includes userId, planId, planName in session
4. **Price Configuration**: Uses constant €29.00 (configurable in future)
5. **Return URLs**: Configurable cancel URL, fixed success URL
6. **Pending Payment**: Creates Payment record with 'pending' status
7. **Spanish Error Messages**: Consistent with existing codebase

## Stripe Checkout Flow

```
Client                          Server                          Stripe
  |                               |                                |
  |--- POST /create-checkout ---> |                                |
  |     { planId, returnUrl }     |                                |
  |                               |--- Create Checkout Session --->|
  |                               |                                |
  |                               |<--- Return checkout URL -------|
  |                               |     + sessionId                |
  |                               |                                |
  |<--- Return checkoutUrl ------|                                |
  |     + sessionId               |                                |
  |                               |                                |
  |--- Redirect to checkoutUrl -->|                                |
  |                               |                                |
  |                               |                           [User pays]
  |                               |                                |
  |                               |<--- Webhook: session.completed|
  |                               |     Update Payment status      |
  |                               |                                |
  |<--- Redirect to success URL --|                                |
```
