# Plan 10-03: PaymentStatusBadge Component

**Wave:** 2 - UI Components

## Overview

Create a reusable badge component that displays payment status with appropriate colors, icons, and labels. Follows the exact pattern of `QRStatusBadge.vue` for consistency.

## Requirements

Covered: STAT-03 (Visual status badges display)

## Objective

Create `app/components/payment/PaymentStatusBadge.vue` that:
1. Displays payment status with visual badge
2. Shows appropriate color (green/orange/red/gray) based on status
3. Shows appropriate icon for each status
4. Uses Spanish labels (Pagado/Pendiente/Fallido/Sin pago)
5. Follows QRStatusBadge pattern for consistency
6. Supports props for size and variant customization

## Success Criteria

1. Component renders UBadge with correct color for each status
2. Component shows correct icon for each status
3. Component shows correct Spanish label for each status
4. Component follows QRStatusBadge pattern exactly
5. Component is reusable across the application

## Implementation

### File: `app/components/payment/PaymentStatusBadge.vue`

```vue
<template>
  <UBadge
    :color="badgeColor"
    :variant="variant"
    :icon="badgeIcon"
    :size="size"
    :class="customClass"
  >
    {{ badgeLabel }}
  </UBadge>
</template>

<script setup lang="ts">
import { computed } from 'vue'

/**
 * PaymentStatusBadge Component
 *
 * Displays a badge showing the payment status
 * with appropriate color, icon, and label.
 *
 * @props {PaymentStatus} status - The payment status
 * @props {string} size - Badge size ('xs' | 'sm' | 'md' | 'lg' | 'xl')
 * @props {string} variant - Badge variant ('solid' | 'outline' | 'soft' | 'subtle')
 * @props {string} customClass - Additional CSS classes
 *
 * @example
 * <PaymentStatusBadge status="succeeded" />
 * <PaymentStatusBadge status="pending" size="sm" variant="outline" />
 */

export type PaymentStatus = 'none' | 'pending' | 'succeeded' | 'failed' | 'canceled' | 'processing'

interface Props {
  status?: PaymentStatus | null
  size?: 'xs' | 'sm' | 'md' | 'lg' | 'xl'
  variant?: 'solid' | 'outline' | 'soft' | 'subtle'
  customClass?: string
}

const props = withDefaults(defineProps<Props>(), {
  status: null,
  size: 'md',
  variant: 'soft',
  customClass: ''
})

/**
 * Badge color based on payment status
 *
 * - succeeded: green (success)
 * - pending: orange (warning)
 * - processing: blue (info)
 * - failed/canceled: red (error)
 * - none: gray (neutral)
 */
const badgeColor = computed<'success' | 'error' | 'warning' | 'info' | 'gray'>(() => {
  switch (props.status) {
    case 'succeeded':
      return 'success'
    case 'pending':
    case 'processing':
      return 'warning'
    case 'failed':
    case 'canceled':
      return 'error'
    default:
      return 'gray'
  }
})

/**
 * Badge label based on payment status (Spanish)
 */
const badgeLabel = computed<string>(() => {
  switch (props.status) {
    case 'succeeded':
      return 'Pagado'
    case 'pending':
      return 'Pendiente'
    case 'processing':
      return 'Procesando'
    case 'failed':
      return 'Fallido'
    case 'canceled':
      return 'Cancelado'
    default:
      return 'Sin pago'
  }
})

/**
 * Badge icon based on payment status
 */
const badgeIcon = computed<string>(() => {
  switch (props.status) {
    case 'succeeded':
      return 'i-heroicons-check-circle'
    case 'pending':
    case 'processing':
      return 'i-heroicons-clock'
    case 'failed':
    case 'canceled':
      return 'i-heroicons-x-circle'
    default:
      return 'i-heroicons-credit-card'
  }
})
</script>
```

### Key Implementation Notes

1. **Pattern Consistency**: Exact copy of QRStatusBadge structure
2. **Spanish Labels**: All user-facing text in Spanish
3. **Nuxt UI v4**: Uses UBadge with correct color values
4. **TypeScript**: Proper type exports for reusable usage
5. **Status Coverage**: Handles all payment statuses from Payment model
6. **Icon Mapping**: Heroicons for consistent iconography
