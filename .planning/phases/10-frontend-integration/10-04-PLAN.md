# Plan 10-04: Payment Page with Stripe Checkout

**Phase:** 10 Frontend Integration
**Wave:** 2 - UI Components
**Created:** 2026-01-27
**Status:** Pending

## Overview

Create a protected payment page that initiates the Stripe Checkout flow. The page validates parameters, creates a checkout session via API, and redirects the user to the hosted Stripe Checkout.

## Requirements

Covered:
- PAY-03: Embedded Stripe Checkout session for pay-per-plan transaction
- PAY-05: Success URL redirects back to plan page

## Objective

Create `app/pages/protected/payment/index.vue` that:
1. Reads query params (planId, returnUrl)
2. Validates plan exists and user has access
3. Shows loading state while creating checkout session
4. Calls API to create Stripe Checkout session
5. Redirects to Stripe Checkout URL using window.location
6. Handles errors gracefully with user-friendly messages
7. Provides back button to return to plan

## Success Criteria

1. Page loads and validates query parameters
2. Page calls `/api/payments/create-checkout` on mount
3. Page shows loading state during API call
4. Page redirects to Stripe Checkout URL when received
5. Page handles API errors with clear error message
6. Page provides back button to return to plan
7. Page uses Nuxt UI v4 components for consistent styling

## Implementation

### File: `app/pages/protected/payment/index.vue`

```vue
<script setup lang="ts">
import { onMounted, ref } from 'vue'

/**
 * Payment Page
 *
 * Initiates Stripe Checkout flow for plan payment.
 *
 * Query Parameters:
 * - planId: string (required) - The plan ID to pay for
 * - returnUrl: string (optional) - URL to return to if payment is canceled
 *
 * Flow:
 * 1. Validate planId and returnUrl from query params
 * 2. Show loading state
 * 3. Call /api/payments/create-checkout
 * 4. Redirect to Stripe Checkout URL
 * 5. Handle errors with user-friendly message
 */

const route = useRoute()
const router = useRouter()
const toast = useToast()

// Query parameters
const planId = ref<string>('')
const returnUrl = ref<string>('/protected/planes')

// State
const isLoading = ref(true)
const error = ref<string | null>(null)
const planName = ref<string>('')

onMounted(async () => {
  // Get query parameters
  const queryPlanId = route.query.planId as string
  const queryReturnUrl = route.query.returnUrl as string

  if (!queryPlanId) {
    error.value = 'No se especificó el plan a pagar'
    isLoading.value = false
    return
  }

  planId.value = queryPlanId
  if (queryReturnUrl) {
    returnUrl.value = decodeURIComponent(queryReturnUrl)
  }

  try {
    // Create checkout session
    const response = await $fetch('/api/payments/create-checkout', {
      method: 'POST',
      body: {
        planId: planId.value,
        returnUrl: returnUrl.value
      }
    })

    // Redirect to Stripe Checkout
    if (response.checkoutUrl) {
      window.location.href = response.checkoutUrl
    } else {
      throw new Error('No se recibió URL de pago')
    }
  } catch (err: any) {
    console.error('Error creating checkout session:', err)
    isLoading.value = false

    // Handle specific error messages
    if (err.statusMessage) {
      error.value = err.statusMessage
    } else if (err.data?.message) {
      error.value = err.data.message
    } else {
      error.value = 'Error al iniciar el proceso de pago'
    }

    // Show toast notification
    toast.add({
      title: 'Error',
      description: error.value,
      color: 'red',
      icon: 'i-heroicons-exclamation-triangle'
    })
  }
})

// Navigate back to plan
const goBack = () => {
  if (planId.value) {
    router.push(`/protected/planes/${planId.value}`)
  } else {
    router.push('/protected/planes')
  }
}
</script>

<template>
  <div class="min-h-screen bg-gray-50 dark:bg-gray-900 py-12 px-4">
    <div class="max-w-md mx-auto">
      <!-- Loading State -->
      <UCard v-if="isLoading && !error">
        <div class="text-center py-8">
          <UIcon
            name="i-heroicons-arrow-path"
            class="w-12 h-12 mx-auto mb-4 text-primary-500 animate-spin"
          />
          <h2 class="text-xl font-semibold text-gray-900 dark:text-white mb-2">
            Preparando pago...
          </h2>
          <p class="text-gray-600 dark:text-gray-400">
            Estás siendo redirigido a la pasarela de pago segura.
          </p>
        </div>
      </UCard>

      <!-- Error State -->
      <UCard v-else-if="error">
        <div class="text-center py-8">
          <UIcon
            name="i-heroicons-exclamation-triangle"
            class="w-12 h-12 mx-auto mb-4 text-red-500"
          />
          <h2 class="text-xl font-semibold text-gray-900 dark:text-white mb-2">
            Error al iniciar el pago
          </h2>
          <p class="text-gray-600 dark:text-gray-400 mb-6">
            {{ error }}
          </p>
          <UButton
            color="primary"
            variant="solid"
            icon="i-heroicons-arrow-left"
            @click="goBack"
          >
            Volver al plan
          </UButton>
        </div>
      </UCard>
    </div>
  </div>
</template>
```

### Key Implementation Notes

1. **Full Page Redirect**: Uses `window.location.href` for Stripe Checkout redirect
2. **Loading State**: Shows spinner during API call
3. **Error Handling**: Catches API errors and shows user-friendly message
4. **Back Button**: Allows user to return to plan if needed
5. **Query Params**: Reads planId and returnUrl from URL
6. **Toast Notification**: Uses Nuxt UI toast for error feedback
7. **Spanish Text**: All user-facing text in Spanish
8. **Nuxt UI v4**: Uses UCard, UIcon, UButton for consistent styling

### URL Structure

**Payment Trigger (from usePaymentGuard):**
```
/payment?planId=abc123&returnUrl=%2Fprotected%2Fplanes%2Fabc123%2Fprint
```

**After Payment Completion:**
```
Success: /protected/planes/abc123/print?payment=success
Cancel: /protected/planes/abc123 (or custom returnUrl)
```

## User Flow

```
1. User clicks "Print" button
   ↓
2. usePaymentGuard.checkPayment() returns false
   ↓
3. usePaymentGuard navigates to /payment?planId=xxx&returnUrl=yyy
   ↓
4. Payment page loads, shows loading spinner
   ↓
5. Payment page calls /api/payments/create-checkout
   ↓
6. API creates Stripe Checkout session
   ↓
7. Payment page redirects to Stripe Checkout URL
   ↓
8. User enters payment details on Stripe
   ↓
9. Stripe redirects to success URL (/protected/planes/xxx/print?payment=success)
```
