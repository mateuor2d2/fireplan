# Plan 10-06: Return URL Handling and Post-Payment Flow

**Phase:** 10 Frontend Integration
**Wave:** 3 - Integration
**Created:** 2026-01-27
**Status:** Pending

## Overview

Modify the print page to handle successful payment return from Stripe Checkout. When the user returns with `?payment=success`, the page should refresh payment status, show confirmation, and enable PDF generation.

## Requirements

Covered:
- PAY-05: Success URL redirects back to plan page to trigger PDF generation after payment

## Objective

Modify `app/pages/protected/planes/[id]/print.vue` to:
1. Detect `?payment=success` query parameter on mount
2. Refresh payment status via `usePaymentGuard.refreshPaymentStatus()`
3. Show success message/toast when payment succeeded
4. Update payment badge to show "Pagado" status
5. Enable PDF generation automatically or prompt user
6. Handle edge cases (payment still pending, failed payment)
7. Use `PaymentStatusBadge` component to display status

## Success Criteria

1. Print page detects `?payment=success` query parameter
2. Page refreshes payment status from API
3. Page shows success message when payment succeeded
4. Payment badge updates to show "Pagado" status
5. PDF generation becomes enabled after successful payment
6. Page handles edge cases gracefully (pending, failed)
7. Page uses PaymentStatusBadge component for status display

### File: `app/pages/protected/planes/[id]/print.vue` (MODIFY EXISTING)

Add payment success handling to the existing print page:

```vue
<script setup lang="ts">
import { onMounted, ref, computed } from 'vue'
import PaymentStatusBadge from '~/components/payment/PaymentStatusBadge.vue'

/**
 * Print Page
 *
 * Handles PDF generation for safety plans with payment enforcement.
 *
 * Query Parameters:
 * - payment=success: User returned from successful Stripe Checkout
 */

const route = useRoute()
const router = useRouter()
const toast = useToast()

const planId = route.params.id as string
const paymentSuccess = ref(false)
const isRefreshingPayment = ref(false)

// Use payment guard composable
const {
  hasPayment,
  paymentStatus,
  isLoading: isCheckingPayment,
  error: paymentError,
  checkPayment,
  refreshPaymentStatus
} = usePaymentGuard()

// Check if user returned from successful payment
const returnedFromPayment = computed(() => {
  return route.query.payment === 'success'
})

onMounted(async () => {
  // Initial payment check
  await checkPayment(planId)

  // If user returned from payment, refresh status and show confirmation
  if (returnedFromPayment.value) {
    paymentSuccess.value = true
    isRefreshingPayment.value = true

    // Refresh payment status to get latest state
    await refreshPaymentStatus(planId)

    isRefreshingPayment.value = false

    // Show success message if payment succeeded
    if (hasPayment.value) {
      toast.add({
        title: '¡Pago completado!',
        description: 'Ya puedes generar el PDF de tu plan de seguridad.',
        color: 'green',
        icon: 'i-heroicons-check-circle',
        timeout: 5000
      })

      paymentSuccess.value = true
    } else {
      // Payment might still be processing
      if (paymentStatus.value === 'pending' || paymentStatus.value === 'processing') {
        toast.add({
          title: 'Pago en proceso',
          description: 'Tu pago está siendo procesado. Por favor, espera unos segundos.',
          color: 'orange',
          icon: 'i-heroicons-clock',
          timeout: 5000
        })

        // Poll for payment status every 3 seconds
        const pollInterval = setInterval(async () => {
          await refreshPaymentStatus(planId)

          if (hasPayment.value || paymentStatus.value === 'failed') {
            clearInterval(pollInterval)

            if (hasPayment.value) {
              toast.add({
                title: '¡Pago completado!',
                description: 'Ya puedes generar el PDF de tu plan de seguridad.',
                color: 'green',
                icon: 'i-heroicons-check-circle'
              })
              paymentSuccess.value = true
            } else if (paymentStatus.value === 'failed') {
              toast.add({
                title: 'Pago fallido',
                description: 'El pago no pudo ser procesado. Por favor, inténtalo de nuevo.',
                color: 'red',
                icon: 'i-heroicons-x-circle'
              })
            }
          }
        }, 3000)

        // Clear interval after 30 seconds
        setTimeout(() => clearInterval(pollInterval), 30000)
      } else if (paymentStatus.value === 'failed') {
        toast.add({
          title: 'Pago fallido',
          description: 'El pago no pudo ser procesado. Por favor, inténtalo de nuevo.',
          color: 'red',
          icon: 'i-heroicons-x-circle'
        })
      }
    }
  }
})

// Generate PDF function
const generatePdf = async () => {
  if (!hasPayment.value) {
    // Trigger payment flow
    await requirePaymentOrTrigger(planId)
    return
  }

  // Generate PDF
  try {
    isLoadingPdf.value = true
    const blob = await $fetch(`/api/planes/${planId}/generate-pdf`, {
      responseType: 'blob'
    })

    // Download PDF
    const url = window.URL.createObjectURL(blob as Blob)
    const a = document.createElement('a')
    a.href = url
    a.download = `plan-seguridad-${planId}.pdf`
    a.click()
    window.URL.revokeObjectURL(url)

    toast.add({
      title: 'PDF generado',
      description: 'Tu plan de seguridad ha sido generado correctamente.',
      color: 'green',
      icon: 'i-heroicons-document-text'
    })
  } catch (err: any) {
    toast.add({
      title: 'Error',
      description: err.data?.statusMessage || 'Error al generar el PDF',
      color: 'red',
      icon: 'i-heroicons-exclamation-triangle'
    })
  } finally {
    isLoadingPdf.value = false
  }
}
</script>

<template>
  <div class="container mx-auto px-4 py-8">
    <!-- Page Header -->
    <div class="mb-6">
      <h1 class="text-2xl font-bold text-gray-900 dark:text-white">
        Generar PDF del Plan
      </h1>
      <p class="text-gray-600 dark:text-gray-400">
        Genera el documento PDF de tu plan de seguridad.
      </p>
    </div>

    <!-- Payment Status Card -->
    <UCard class="mb-6">
      <template #header>
        <h2 class="text-lg font-semibold">Estado del Pago</h2>
      </template>

      <div class="flex items-center justify-between">
        <div class="flex items-center gap-4">
          <!-- Payment Status Badge -->
          <PaymentStatusBadge
            :status="paymentStatus"
            size="lg"
          />

          <!-- Loading State -->
          <USpinner v-if="isRefreshingPayment" size="sm" />
        </div>

        <!-- Payment Status Text -->
        <div class="text-right">
          <p v-if="hasPayment" class="text-sm text-green-600 dark:text-green-400">
            ✓ El pago ha sido completado
          </p>
          <p v-else-if="paymentStatus === 'pending'" class="text-sm text-orange-600 dark:text-orange-400">
            ⏳ El pago está siendo procesado
          </p>
          <p v-else class="text-sm text-gray-600 dark:text-gray-400">
            Se requiere pago para generar el PDF
          </p>
        </div>
      </div>
    </UCard>

    <!-- Success Message (when returned from payment) -->
    <UAlert
      v-if="paymentSuccess && hasPayment"
      color="green"
      icon="i-heroicons-check-circle"
      title="¡Pago completado!"
      class="mb-6"
    >
      Ya puedes generar el PDF de tu plan de seguridad.
    </UAlert>

    <!-- Generate PDF Button -->
    <UCard>
      <template #header>
        <h2 class="text-lg font-semibold">Generar PDF</h2>
      </template>

      <div class="flex flex-col gap-4">
        <p class="text-gray-600 dark:text-gray-400">
          Haz clic en el botón para generar el documento PDF del plan de seguridad.
        </p>

        <UButton
          :color="hasPayment ? 'primary' : 'gray'"
          variant="solid"
          size="lg"
          :loading="isLoadingPdf"
          :disabled="!hasPayment && !isCheckingPayment"
          icon="i-heroicons-document-arrow-down"
          @click="generatePdf"
        >
          <template v-if="!hasPayment">
            Pagar €29 y Generar PDF
          </template>
          <template v-else>
            Generar PDF
          </template>
        </UButton>

        <p v-if="paymentError" class="text-sm text-red-600 dark:text-red-400">
          {{ paymentError }}
        </p>
      </div>
    </UCard>
  </div>
</template>
```

### Key Implementation Notes

1. **Query Param Detection**: Checks `route.query.payment === 'success'`
2. **Status Refresh**: Calls `refreshPaymentStatus()` on return from payment
3. **Success Message**: Shows toast notification when payment succeeded
4. **Polling**: Polls for payment status if still pending (3s intervals, 30s timeout)
5. **Payment Badge**: Uses `PaymentStatusBadge` component for visual feedback
6. **Button State**: Changes button text based on payment status
7. **Edge Cases**: Handles pending, processing, failed states
8. **Spanish Messages**: All user-facing text in Spanish

### Payment Flow

```
Stripe Checkout Success
        ↓
Redirect to: /protected/planes/123/print?payment=success
        ↓
Print page detects payment=success
        ↓
Refresh payment status via API
        ↓
    ┌───────────────┴───────────────┐
    │                               │
Payment succeeded            Payment pending/processing
    │                               │
Show success toast           Show processing toast
    │                               │
Enable PDF button            Poll for status updates
    │                          (every 3s, max 30s)
    │                               │
    │                          ┌─────┴─────┐
    │                          │           │
    │                   Payment succeeded  Payment failed
    │                          │           │
    │                   Show success     Show error
    │                   Enable PDF       Prompt retry
    │                          │           │
    └──────────────────────────┴───────────┘
```
