**Output:** Two API endpoints (pause/resume) that modify subscription state using Stripe's pause_collection feature with database synchronization.

<task type="auto">
  <name>Create pause subscription endpoint</name>
  <files>server/api/planes/[id]/subscription/pause.post.ts</files>
  <action>
Create POST endpoint at `/api/planes/[id]/subscription/pause` that:

1. **Authentication check**: Verify event.context.user exists, return 401 if not

2. **Parse request body** with Zod schema:
   ```typescript
   const pauseSchema = z.object({
     resumeDate: z.string().optional(), // ISO date string or undefined for indefinite
     indefinite: z.boolean().default(false)
   })
   ```

3. **Validate plan ownership**:
   - Extract userId from authenticated user
   - Extract planId from route params (event.context.params.id)
   - Query Planes: { _id: planId, createdBy: userId }
   - Return 404 if plan not found or not owned by user

4. **Validate subscription exists**:
   - Query Subscription: { planId, userId }
   - Return 404 if no subscription found
   - Return 400 if subscription.status is not 'active' (cannot pause paused/canceled)

5. **Calculate resume date**:
   - If indefinite=true: set resumesAt = undefined
   - If resumeDate provided: parse ISO string to Date
   - If neither: default to indefinite=true

6. **Call Stripe API**:
   - Import updateQrSubscription from server/utils/stripe.ts
   - Call updateQrSubscription({
       subscriptionId: subscription.stripeSubscriptionId,
       updates: {
         pause_collection: {
           behavior: 'keep_as_draft',
           resumes_at: indefinite ? undefined : resumeDate
         }
       }
     })

7. **Update database record**:
   - Update Subscription document:
     - pauseCollection.behavior = 'keep_as_draft'
     - pauseCollection.resumesAt = resumesAt (Date or undefined)
     - status = 'paused'
   - Save document

8. **Return success response**:
   ```typescript
   return {
     success: true,
     data: {
       subscriptionId: subscription._id,
       status: 'paused',
       resumesAt: subscription.pauseCollection.resumesAt
     }
   }
   ```

9. **Error handling**:
   - Wrap in try/catch
   - Handle Zod validation errors (400)
   - Handle Stripe API errors (log and return 500 with message)
   - Return existing error objects if statusCode exists

**IMPORTANT:** Follow existing pattern from server/api/subscriptions/post.ts for authentication, plan ownership check, and error handling.
