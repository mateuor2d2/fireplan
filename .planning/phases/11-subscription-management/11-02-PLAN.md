<task type="auto">
  <name>Create cancel subscription endpoint</name>
  <files>server/api/planes/[id]/subscription/cancel.post.ts</files>
  <action>
Create POST endpoint at `/api/planes/[id]/subscription/cancel` that:

1. **Authentication check**: Verify event.context.user exists, return 401 if not

2. **Validate plan ownership**:
   - Extract userId from authenticated user
   - Extract planId from route params
   - Query Planes: { _id: planId, createdBy: userId }
   - Return 404 if plan not found or not owned

3. **Validate subscription state**:
   - Query Subscription: { planId, userId }
   - Return 404 if no subscription found
   - Return 400 if status is 'canceled' or 'expired' (cannot cancel canceled)
   - Return 400 if cancelAtPeriodEnd is already true

4. **Call Stripe API**:
   - Import cancelQrSubscription from server/utils/stripe.ts
   - Call cancelQrSubscription({
       subscriptionId: subscription.stripeSubscriptionId,
       cancelAtPeriodEnd: true // Graceful cancellation at period end
     })

5. **Update database record**:
   - Update Subscription document:
     - cancelAtPeriodEnd = true
     - canceledAt = new Date()
     - Keep status as 'active' (will update via webhook when period ends)
     - Set expiresAt = currentPeriodEnd (for UI display)
   - Save document

6. **Return success response**:
   ```typescript
   return {
     success: true,
     data: {
       subscriptionId: subscription._id,
       status: 'active', // Still active until period end
       cancelAtPeriodEnd: true,
       expiresAt: subscription.currentPeriodEnd
     }
   }
   ```

7. **Error handling**:
   - Same pattern as pause/resume endpoints
   - Handle Zod validation errors
   - Handle Stripe API errors
   - Return existing errors

**IMPORTANT:** This implements "period end" cancellation as specified in CONTEXT.md. Users keep access until currentPeriodEnd. No request body needed.

<task type="auto">
  <name>Create re-subscription endpoint</name>
  <files>server/api/planes/[id]/subscription/resubscribe.post.ts</files>
  <action>
Create POST endpoint at `/api/planes/[id]/subscription/resubscribe` that creates a new Stripe Checkout session for subscription:

1. **Authentication check**: Verify event.context.user exists, return 401 if not

2. **Validate plan ownership**:
   - Extract userId from authenticated user
   - Extract planId from route params
   - Query Planes: { _id: planId, createdBy: userId }
   - Return 404 if plan not found or not owned

3. **Get user information**:
   - Query User.findById(userId)
   - Return 404 if user not found

4. **Validate re-subscription eligibility**:
   - Query Subscription: { planId, userId }
   - Allow re-subscription if:
     - No previous subscription exists (new sub)
     - Previous subscription is 'canceled' or 'expired'
   - Return 400 if previous subscription is 'active' or 'paused' (must cancel first)

5. **Get or create Stripe customer**:
   - Import stripe from server/utils/stripe.ts
   - List customers by email: stripe.customers.list({ email: user.email, limit: 1 })
   - Use existing customer if found, else create new:
     ```typescript
     stripeCustomer = await stripe.customers.create({
       email: user.email,
       name: user.name || user.matriz_nombre,
       metadata: { userId }
     })
     ```

6. **Create Stripe Checkout session**:
   - Import stripe from server/utils/stripe.ts
   - Call stripe.checkout.sessions.create({
       customer: stripeCustomer.id,
       mode: 'subscription',
       payment_method_types: ['card'],
       line_items: [{
         price: process.env.STRIPE_MONTHLY_PRICE_ID, // Default to monthly
         quantity: 1
       }],
       success_url: `${process.env.NUXT_PUBLIC_SITE_URL}/protected/planes/${planId}?subscription=new`,
       cancel_url: `${process.env.NUXT_PUBLIC_SITE_URL}/protected/planes/${planId}`,
       metadata: {
         userId,
         planId,
         planName: plan.nom_obra,
         resubscription: 'true'
       },
       subscription_data: {
         metadata: {
           userId,
           planId,
           planName: plan.nom_obra,
           resubscription: 'true'
         }
       }
     })

7. **Return success response**:
   ```typescript
   return {
     success: true,
     data: {
       checkoutUrl: session.url,
       sessionId: session.id
     }
   }
   ```

8. **Error handling**:
   - Same pattern as other endpoints
   - Handle Stripe API errors
   - Return existing errors

**IMPORTANT:** Follow existing pattern from server/api/payments/create-checkout.post.ts (Phase 10 Plan 02). Re-subscription uses Stripe Checkout for payment method collection and subscription creation. Webhook will create new Subscription record.
