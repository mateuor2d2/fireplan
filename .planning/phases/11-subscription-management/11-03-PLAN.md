  <action>
Create POST endpoint at `/api/planes/[id]/subscription/portal` that:

1. **Authentication check**: Verify event.context.user exists, return 401 if not

2. **Validate plan ownership**:
   - Extract userId from authenticated user
   - Extract planId from route params
   - Query Planes: { _id: planId, createdBy: userId }
   - Return 404 if plan not found or not owned

3. **Validate subscription exists**:
   - Query Subscription: { planId, userId }
   - Return 404 if no subscription found

4. **Get Stripe customer ID**:
   - Extract stripeCustomerId from subscription document
   - Return 400 if stripeCustomerId is missing

5. **Create Billing Portal session**:
   - Import stripe from server/utils/stripe.ts
   - Build return URL: `${process.env.NUXT_PUBLIC_SITE_URL}/protected/planes/${planId}?portal_return=true`
   - Call stripe.billingPortal.sessions.create({
       customer: stripeCustomerId,
       return_url: returnUrl,
       // Optional: Restrict portal to specific subscription
       subscription: subscription.stripeSubscriptionId // Limits portal to this subscription
     })

6. **Return success response**:
   ```typescript
   return {
     success: true,
     data: {
       portalUrl: session.url,
       sessionId: session.id
     }
   }
   ```

7. **Error handling**:
   - Same pattern as other subscription endpoints
   - Handle Stripe API errors (log and return 500)
   - Return existing errors

**IMPORTANT CONTEXT DECISIONS:**
- Portal access is from plan details page (CONTEXT.md: "Location: Plan details page")
- Portal link is in settings/navigation area (CONTEXT.md: "Portal Access: Settings page")
- Return URL includes ?portal_return=true query param for success toast detection
- Subscription-scoped portal (subscription parameter) limits management to this plan's subscription

**STRIPE PORTAL CONFIGURATION:**
Note: User must configure Customer Portal in Stripe Dashboard:
- Products/prices to display
- Allowed updates (pause, cancel, upgrade)
- Branding options
This is documented in user_setup section of frontmatter.
