# GSD Plan: Update Nuxt Config with Nitro Externals + Code Splitting

## Objective
Optimize Nuxt configuration for smaller bundle size and better code splitting

## Problem
- Heavy dependencies bundled unnecessarily
- No code splitting for large vendor chunks
- Missing optimization flags

## Implementation

### Task 1: Update nuxt.config.ts with Nitro optimizations

```typescript
export default defineNuxtConfig({
  // ... existing config ...
  
  nitro: {
    preset: 'vercel',
    
    externals: {
      external: [
        'pdfmake',
        'sharp',
        '@aws-sdk/client-s3',
        '@aws-sdk/s3-request-presigner',
        'mongoose',
        'bcryptjs',
        'jsonwebtoken',
        'stripe',
        'twilio',
        'mailgun.js'
      ],
      inline: [
        // Keep small, frequently used packages inline
      ]
    },
    
    rollupConfig: {
      external: [
        'pdfmake',
        'sharp',
        '@aws-sdk/client-s3',
        '@aws-sdk/s3-request-presigner',
        'mongoose',
        'bcryptjs',
        'jsonwebtoken',
        'stripe',
        'twilio',
        'mailgun.js'
      ],
      output: {
        manualChunks(id) {
          // Split large dependencies into separate chunks
          if (id.includes('node_modules')) {
            if (id.includes('pdfmake') || id.includes('vfs_fonts')) {
              return 'pdf-worker'
            }
            if (id.includes('@aws-sdk')) {
              return 'aws-sdk'
            }
            if (id.includes('mongoose')) {
              return 'mongoose'
            }
            if (id.includes('sharp')) {
              return 'image-processing'
            }
          }
        }
      }
    },
    
    experimental: {
      wasm: false
    },
    
    compressPublicAssets: true
  },
  
  vite: {
    build: {
      sourcemap: false,
      minify: 'terser',
      chunkSizeWarningLimit: 1000,
      rollupOptions: {
        output: {
          manualChunks: {
            'vue-vendor': ['vue', 'vue-router', 'pinia'],
            'ui-vendor': ['@nuxt/ui', 'reka-ui'],
            'utils-vendor': ['zod', 'slugify']
          }
        }
      }
    }
  },
  
  features: {
    inlineStyles: true
  }
})
```

### Task 2: Add build memory optimization script
Update package.json scripts:
```json
{
  "scripts": {
    "build": "NODE_OPTIONS='--max-old-space-size=4096' nuxt build",
    "build:ci": "NODE_OPTIONS='--max-old-space-size=2048' nuxt build"
  }
}
```

## Success Criteria
- [ ] Nitro externals configured for all heavy deps
- [ ] Code splitting configured for vendors
- [ ] Source maps disabled in production
- [ ] .output/server < 40MB
- [ ] Build completes with 4GB memory limit
- [ ] No runtime errors

## Expected Savings
- **Server bundle**: ~20-30MB reduction
- **Build memory**: From 8GB to 4GB requirement
- **Risk**: Medium (config changes need testing)
