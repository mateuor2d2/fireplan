# Plan 15-01: Review and Fix Detalles Gráficos PDF Generation

**Phase:** 15-detalles-graficos-review  
**Goal:** Identify and fix issues with detalles gráficos (graphical details) not printing correctly in PDFs  
**Type:** Bug fix / Quality assurance  
**Estimated Time:** 2-3 hours  
**Depends On:** Phase 14 (Deployment Optimization) - COMPLETE

---

## Current State

Detalles gráficos is a feature that allows users to attach images to safety plans. These images should appear in the generated PDF. The system involves:

- **Frontend:** `DetallesGraficosComponent.vue`, `PlanDetallesGraficosManager.vue`
- **Backend:** `generate-pdf.get.ts`, `templateImageProcessor.ts`
- **Data Flow:** Images stored in S3 → converted to base64 for PDF generation

### Known Issues

Based on user report: "detalles graficos has some issues at time to print"

**Suspected Problem Areas:**
1. Image conversion from S3 URLs to base64 may fail silently
2. PDF generation may skip detalles graficos section if images not processed
3. Template variables (`{{detalles_graficos}}`) may not be replaced correctly
4. Image markers in HTML may not be converted to pdfmake image objects

### Step 2: Check Image Data Flow

**Verify Components:**

1. **Image Upload (S3)**
   - File: `server/api/s3/upload.post.ts`
   - Check: Images upload successfully to S3
   - Verify: URLs are accessible and not expired

2. **Image Storage (Plan Model)**
   - Field: `plan.det_graf` array
   - Check: Images stored with correct metadata (url, name, type)
   - Verify: URLs are preserved in database

3. **Image Preprocessing (Template)**
   - File: `server/utils/templateImageProcessor.ts`
   - Function: `preprocessTemplateImages()`
   - Check: S3 URLs detected and converted to base64
   - Verify: `detalles_graficos_processed` variable created

4. **PDF Generation Pipeline**
   - File: `server/api/planes/[id]/generate-pdf.get.ts`
   - Class: `ImageProcessingPipeline`
   - Function: `processImagesFromTemplate()`
   - Check: Images processed from `templateData.det_graf`
   - Verify: Image placeholders cached correctly

5. **HTML to PDF Conversion**
   - File: `server/api/planes/[id]/generate-pdf.get.ts`
   - Function: `processDetallesGraficosSection()`
   - Check: HTML elements processed into pdfmake objects
   - Verify: Image placeholders replaced with actual image data

### Step 3: Identify Root Cause

**Common Issues to Check:**

| Issue | Symptom | Likely Cause | Fix |
|-------|---------|--------------|-----|
| Images not appearing | Empty detalles graficos section | Template variable not replaced | Check `{{detalles_graficos}}` replacement |
| Broken image icons | Red X or placeholder | Base64 conversion failed | Check S3 URL accessibility and conversion |
| Images cut off | Partial image display | Width/height not calculated | Check image dimension processing |
| Missing entirely | No detalles graficos section | Section not detected in HTML | Check section type detection |
| Quality issues | Pixelated images | Compression or sizing | Check image processing parameters |
