# Plan 16-01: Restore CSP Headers with Stripe Domain Whitelisting

## Goal
Restore and enhance Content Security Policy (CSP) headers in nuxt.config.ts with comprehensive Stripe domain whitelisting for PCI compliance.

## Context
Phase 9.3 previously implemented CSP headers, but they appear to have been lost during subsequent refactoring. This plan restores the CSP configuration with all required Stripe domains for secure payment processing.

## must_haves

1. CSP header is configured in nuxt.config.ts routeRules
2. All Stripe domains are whitelisted (js.stripe.com, checkout.stripe.com, api.stripe.com, *.stripe.com, *.stripe.network)
3. Production environment includes upgrade-insecure-requests directive
4. Build completes without CSP-related errors
5. Configuration follows Phase 9.3 patterns but improves upon them

## Verification Criteria

- CSP header present in HTTP response headers
- Stripe.js loads without CSP violations in browser console
- Stripe Checkout iframe loads correctly
- No console errors related to CSP
- Headers match or exceed Phase 9.3 implementation

## Implementation Notes

Add to nitro section in nuxt.config.ts:

```typescript
routeRules: {
  '/**': {
    headers: {
      'Content-Security-Policy': [
        "default-src 'self'",
        "script-src 'self' 'unsafe-inline' js.stripe.com",
        "style-src 'self' 'unsafe-inline'",
        "frame-src 'self' js.stripe.com checkout.stripe.com",
        "form-src 'self' checkout.stripe.com",
        "connect-src 'self' api.stripe.com",
        "img-src 'self' data: *.stripe.com *.stripe.network",
        "font-src 'self'",
        "object-src 'none'",
        "base-uri 'self'",
        "frame-ancestors 'self'",
        process.env.NODE_ENV === 'production' ? "upgrade-insecure-requests" : ""
      ].filter(Boolean).join('; ')
    }
  }
}
```
