# Plan 16-02: Implement HSTS and Additional Security Headers

## Goal
Add Strict-Transport-Security (HSTS) header for HTTPS enforcement and implement additional security headers (X-Frame-Options, X-Content-Type-Options, X-XSS-Protection, Referrer-Policy, Permissions-Policy) to complete the security headers suite.

## Context
HTTPS enforcement via HSTS is required for PCI compliance. Additional security headers provide defense-in-depth against common web attacks (clickjacking, MIME sniffing, XSS). These headers were part of Phase 9.3 but need to be restored.

## Tasks

<task>
<name>Add HSTS header for HTTPS enforcement</name>
<description>Add Strict-Transport-Security header to routeRules, configured only for production environment to avoid development issues. Use max-age of 31536000 seconds (1 year) with includeSubDomains and preload directives.</description>
<validation>
- [ ] HSTS header added to routeRules headers
- [ ] Header only applies in production (NODE_ENV === 'production')

</validation>
</task>

## must_haves

1. Strict-Transport-Security header is configured (production only)
2. X-Frame-Options: SAMEORIGIN is set
3. X-Content-Type-Options: nosniff is set
4. X-XSS-Protection: 1; mode=block is set
5. Referrer-Policy: strict-origin-when-cross-origin is set
6. Permissions-Policy restricts camera, microphone, and geolocation to self
7. All headers work correctly in both development and production

## Verification Criteria

- All security headers present in HTTP response
- HSTS header only appears in production (check via NODE_ENV)
- Headers match Phase 9.3 implementation patterns
- No security header warnings in browser console
- Application functionality remains intact

## Implementation Notes

Add to existing routeRules in nuxt.config.ts (building on Plan 16-01):

```typescript
routeRules: {
  '/**': {
    headers: {
      // ... CSP header from Plan 16-01 ...
      'X-Frame-Options': 'SAMEORIGIN',
      'X-Content-Type-Options': 'nosniff',
      'X-XSS-Protection': '1; mode=block',
      'Referrer-Policy': 'strict-origin-when-cross-origin',
      'Permissions-Policy': 'camera=(self), microphone=(self), geolocation=(self)',
      ...(process.env.NODE_ENV === 'production' && {
        'Strict-Transport-Security': 'max-age=31536000; includeSubDomains; preload'
      })
    }
  }
}
```
