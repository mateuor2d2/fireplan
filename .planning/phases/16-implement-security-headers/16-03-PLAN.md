# Plan 16-03: Security Verification and PCI Documentation

## Goal
Verify all 5 security requirements (SEC-01 through SEC-05) are complete and document PCI DSS SAQ A compliance status for audit trail.

## Context
This verification plan confirms that all security gaps from Phase 09 audit have been closed. SEC-01, SEC-02, and SEC-05 were already implemented in previous phases but need verification. SEC-03 and SEC-04 are being implemented in Plans 16-01 and 16-02.

## Tasks

<task>
<name>Verify Stripe Checkout PCI compliance (SEC-01)</name>
<description>Verify that Stripe Checkout is properly implemented and provides PCI DSS SAQ A compliance. Confirm no card data touches our servers.</description>
<validation>
- [ ] Stripe Checkout implementation verified in server/api/payments/create-checkout.post.ts
- [ ] Checkout redirects to checkout.stripe.com (hosted by Stripe)
- [ ] No card data collection on our domains
- [ ] PCI SAQ A compliance confirmed
</validation>
</task>

<task>
<name>Verify no card data storage (SEC-02)</name>
<description>Audit all database models and API endpoints to confirm no raw card data (number, CVV, expiration) is stored or processed.</description>
<validation>
- [ ] Payment.ts model reviewed - no card data fields
- [ ] Subscription.ts model reviewed - no card data fields
- [ ] All API endpoints reviewed - no card data handling
- [ ] Only Stripe IDs stored (stripePaymentIntentId, stripeCustomerId, stripeSubscriptionId)
</validation>
</task>

<task>
<name>Verify CSP headers in production (SEC-03)</name>
<description>Build and test the application to verify CSP headers are properly set and Stripe domains are whitelisted. Test Stripe functionality.</description>
<validation>
- [ ] Run bun build successfully
- [ ] Verify CSP header present in HTTP response
- [ ] Confirm js.stripe.com is in script-src
- [ ] Confirm checkout.stripe.com is in frame-src and form-src
- [ ] Confirm api.stripe.com is in connect-src
- [ ] Test Stripe.js loads without CSP violations
- [ ] Test Stripe Checkout iframe opens successfully
</validation>
</task>

<task>
<name>Verify HTTPS enforcement (SEC-04)</name>
<description>Verify HSTS header is present in production builds and correctly enforces HTTPS connections.</description>
<validation>
- [ ] Build with NODE_ENV=production
- [ ] Verify Strict-Transport-Security header is present
- [ ] Confirm max-age=31536000
- [ ] Confirm includeSubDomains directive
- [ ] Confirm preload directive
- [ ] Development build does NOT include HSTS
</validation>
</task>

<task>
<name>Verify webhook signature verification (SEC-05)</name>
<description>Verify webhook signature verification is mandatory on all Stripe webhook events and properly rejects invalid signatures.</description>
<validation>
- [ ] Review server/api/webhooks/stripe.post.ts
- [ ] Confirm stripe.webhooks.constructEvent is used
- [ ] Verify rawBody is preserved for signature validation
- [ ] Confirm 400 response for invalid signatures
- [ ] All events verified before processing
</validation>
</task>

<task>
<name>Create PCI compliance documentation</name>
<description>Document the PCI DSS SAQ A compliance status covering all security measures. Create summary showing all 5 requirements are met.</description>
<validation>
- [ ] SEC-01 documented: Stripe Checkout for PCI compliance
- [ ] SEC-02 documented: No card data storage
- [ ] SEC-03 documented: CSP headers with Stripe domains
- [ ] SEC-04 documented: HTTPS enforcement via HSTS
- [ ] SEC-05 documented: Webhook signature verification
- [ ] Compliance summary created in planning docs
</validation>
</task>

<task>
<name>Run final security verification</name>
<description>Run complete verification suite including build, type checking, and linting to ensure no security-related regressions.</description>
<validation>
- [ ] bun build completes without errors
- [ ] bun typecheck passes
- [ ] bun lint passes
- [ ] No security-related warnings
- [ ] All tests pass (if applicable)
</validation>
</task>
